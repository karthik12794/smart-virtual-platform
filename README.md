# Smart Virtual Platform (Frontend Demo)

## Overview

**Smart Virtual Platform** is a **frontend-only web application** that simulates a complete **virtual healthcare and banking service system**.  
It addresses the **real-world problem** of **difficulty accessing hospitals, doctors, and bank officials remotely**, providing users with a **seamless, interactive experience**.

This project demonstrates a realistic, functional **demo platform** using **only frontend technologies**, including login, booking, audio/video calls, payments, and digital signatures.

---

## Problem Statement

In today's world, patients often face multiple challenges:

1. Long waiting times at hospitals for OP booking.
2. Difficulty scheduling audio or video consultations with doctors.
3. Limited remote access to bank officials for queries or payment issues.
4. Lack of a unified platform to manage healthcare and financial interactions.

**Smart Virtual Platform** solves these issues by providing a **centralized, virtual solution**.

---

## Solution

The platform offers a **simulated interactive demo** where users can:

- **Login securely** using phone number → OTP → 4-digit PIN.
- **Book OP appointments** with personal details and digital signature.
- **Request audio/video calls** with doctors or bank officials.
- **Simulate payments** via credit/debit card or PhonePe.
- Receive **call confirmations** (Accepted / Denied) for consultations.

---

## Key Features

1. **Login System**
   - Phone number input.
   - OTP verification (simulated).
   - 4-digit security PIN.
   - Protected pages.

2. **Home Dashboard**
   - Mobile-first UI with realistic background.
   - Cards for:
     - Book OP
     - Doctor Call
     - Bank Call
   - Logout option.

3. **OP Booking**
   - Input: Name, Email, Phone, Address.
   - **Digital signature canvas**.
   - Redirect to payment.

4. **Payment Simulation**
   - Options: Card / PhonePe.
   - Demo accepts any input.
   - Confirms payment success.

5. **Audio/Video Call Simulation**
   - Select recipient (doctor/bank manager/cashier).
   - Choose call type (audio/video).
   - Random acceptance for realism.

6. **Frontend-only Implementation**
   - Uses **localStorage** for state management.
   - Fully interactive without backend.
